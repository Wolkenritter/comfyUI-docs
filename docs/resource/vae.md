# VAE 模型文档

VAE（变分自编码器）是图像生成的关键组件。

## 什么是 VAE

VAE 负责编码和解码图像与潜在空间：

- **编码 (Encode)**: 图像 → 潜在空间
- **解码 (Decode)**: 潜在空间 → 图像

### 作用

| 功能 | 说明 |
|------|------|
| 压缩 | 降低计算量 |
| 解码 | 生成最终图像 |
| 质量 | 影响图像细节 |

## VAE 类型

### 标准 VAE

| 模型 | 适用 | 说明 |
|------|------|------|
| vae-ft-ema | SD 1.5 | 官方 EMA VAE |
| vae-ema | SD 1.5 | 稳定版本 |
| kl-f8 | SDXL | SDXL 专用 |

### 专用 VAE

| VAE | 用途 |
|------|------|
| SDXL VAE | SDXL 模型 |
| Clear VAE | 更清晰的结果 |
| MSE VAE | 平滑过渡 |

## 使用方法

### 自动加载

大部分模型内置 VAE，通常自动加载：

```
Checkpoint 模型包含:
- UNet: 主生成网络
- VAE: 编码/解码器
- CLIP: 文本编码器

自动使用模型自带的 VAE
```

### 手动指定

### 手动加载 VAE

```
节点: VAELoader

用法:
1. 加载 VAE 文件
2. 连接到 KSampler 或 Decode 节点
3. 覆盖默认 VAE
```

### 分离 VAE

```
节点: VAEDecode, VAEEncode

说明:
- 可使用与模型不同的 VAE
- 实现特殊效果
```

## VAE 应用场景

### 图像解码

```
潜在空间 → VAE 解码 → 最终图像

控制细节和质量
```

### 图像编码

```
输入图像 → VAE 编码 → 潜在空间

用于图生图和局部重绘
```

### 分块解码

```
大图像 → 分块编码 → 分块解码 → 拼接

降低显存占用
```

## VAE 优化

### 快速解码

```
使用快速 VAE:
- 减少解码步数
- 降低质量（可选）

适合预览
```

### 高质量解码

```
使用完整 VAE:
- 所有解码步骤
- 最佳图像质量

适合最终输出
```

### 分块处理

```
大图像处理:
1. 分块编码
2. 潜在空间处理
3. 分块解码
4. 拼接

节省显存
```

## VAE 问题解决

### 图像模糊

**原因**: VAE 不匹配或损坏

**解决**:
- 使用配套的 VAE
- 重新下载 VAE 文件

### 颜色异常

**原因**: VAE 版本不兼容

**解决**:
- 使用模型配套 VAE
- 尝试其他 VAE

### 显存不足

**解决**:
- 使用分块 VAE 解码
- 减小图像尺寸
- 使用更小的 VAE
